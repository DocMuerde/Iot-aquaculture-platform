# Board v2 – Main Controller Board

## Description
This board implements the main control unit of the IoT aquaculture platform. It is responsible for data acquisition, processing, storage, and communication with external systems.

The board is centered around an ESP32 microcontroller and provides interfaces for sensors, storage, and peripheral expansion modules.

## Features

- **ESP32-based controller**  
  Main processing unit handling:
  - Sensor acquisition
  - Data logging
  - Communication (WiFi / MQTT)
  - System coordination

- **Analog sensor interface (up to 5 channels)**  
  Designed to support analog sensors such as:
  - pH
  - Electrical Conductivity (EC)
  - Dissolved Oxygen (O₂)

  ⚠️ *Currently not used due to nonlinearity issues in the ESP32 ADC, which affect measurement accuracy.*

- **Digital sensors support**  
  - 2 × DHT22 (air temperature and humidity)
  - 2 × DS18B20 (water temperature, 1-Wire protocol)

- **MicroSD interface (2 possible connection points)**  
  Used for:
  - Local data logging
  - Configuration storage (`config.txt`)

- **I2C communication ports (×2)**  
  Used to connect:
  - I2C peripheral board (Board v1)
  - Additional I2C devices if required

- **Digital inputs (×2)**  
  General-purpose inputs (e.g., float switches or auxiliary signals)

- **Inter-board connector**  
  Dedicated connector for communication and integration with Board v1 (I2C interface board)

## Notes

- Originally designed for the Acuacol project, but intended as a reusable core module for the platform.

- Analog sensors must include **galvanic isolation**, implemented through external analog front-end modules.

- Due to ADC limitations of the ESP32, high-precision analog measurements are currently delegated to external ADCs (e.g., ADS1115 on Board v1).

## Future Version

A redesigned version of this board is planned with the following improvements:

- **Replacement of ESP32 with Arduino UNO R4 (RA4M1)**  
  Motivated by:
  - Improved ADC linearity and measurement reliability
  - Better handling of external interrupts (notably those generated by the TCA9555 I/O expander)

- **Integration of cellular communication (SIM module, e.g., SIM900)**  
  - Enables data transmission without relying on WiFi infrastructure
  - Improves deployability in remote or rural environments
